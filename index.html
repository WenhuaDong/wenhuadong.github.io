Introduction

Information sharing can be a key to improve our life if there is a system which can provide a platform to citizens for sharing the knowledge about their living environment and allow them to use the collective information shared by others. It can help citizens in taking decisions whereabouts which depends on the quality of environment of the place.

Tools and Technology Selection

This system can be divided into two different subsystems -

  1. A web application for showing visualizations from collected data and allowing user to share the environmental information (version one).

  2. An android application for supporting those advance features which are limited by the browsers. For example, Augmented Reality (for the future versions).

Requirements for Web Application for PESQoL

This web application is a client server application, in which client is a browser and server is a software written in Java.

Client

Client application will have the responsibility of providing user a way to send environmental and personal information to the server. It also present environmental information in a comprehensible way using Charts, Graphs and Maps to user.

Tools and Technology -

  1. HTML - for designing web pages

  2. CSS - For styling web pages

  3. JavaScript - for manipulating web pages and making them interactive

  4. jQuery - A JavaScript based library to take care of cross browser compatibility

  5. d3.js - D3.js is a JavaScript library for creating graphs, charts and maps.

  6. Weather Underground APIs - This service, provide by the http://www.wunderground.com/, can be used to get the weather information about a place

  7. Google Maps API - Google Maps API can be used to visualize environmental data on maps, and can also be used to find a place using GPS.
